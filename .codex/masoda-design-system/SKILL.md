# Skill — Masoda Design System Guardrail

## Purpose
Use this skill when the task involves visual language, UI consistency, styling choices, section polish, or brand coherence for the Masoda website.

This skill exists to stop the interface from drifting into one of the common failure modes:
- generic template feel,
- over-decorated wellness aesthetic,
- cheap boho or pseudo-esoteric styling,
- cold corporate layout,
- cluttered ecommerce look.

## Brand translation
Masoda should feel:
- warm,
- premium,
- luminous,
- joyful,
- community-oriented,
- creative,
- easy to navigate.

It should not feel:
- loud,
- militant in an overt visual way,
- dark and luxurious in a heavy way,
- childlike,
- hyper-feminised cliché,
- mass ecommerce,
- tech startup.

## Core visual rules
### 1. Colour strategy
Use a restrained palette.
Primary mood should come from:
- warm orange / terracotta / apricot families,
- soft brown / cacao / clay support tones,
- subtle blue as an accent only.

Guidelines:
- large surfaces should remain breathable and calm,
- bright colour should be used to create focal points, not cover everything,
- blue should be a strategic accent, not the dominant page colour,
- avoid saturated green-led wellness clichés,
- avoid black-heavy luxury treatment.

### 2. Logo handling
- Preserve the official logo with halo.
- Do not redesign the type.
- Do not alter the logo colour.
- Give the logo enough whitespace so it feels intentional and premium.

### 3. Typography hierarchy
The interface should feel editorial but easy.
Use a hierarchy that clearly separates:
- brand/promotional headlines,
- section headings,
- support copy,
- utility labels,
- button text.

Rules:
- keep headings short and impactful,
- avoid dense paragraph blocks,
- line length should stay comfortable,
- body text must remain highly readable on mobile,
- do not overuse decorative text treatments.

### 4. Spacing system
Masoda needs breathing room.
Prefer:
- generous section padding,
- distinct vertical rhythm,
- cards and modules with enough internal space,
- obvious separation between content zones.

Avoid:
- cramped compositions,
- too many tiny cards packed together,
- narrow gutters,
- large visual jumps without transitional rhythm.

### 5. Shape language
Prefer soft but controlled geometry.
Good options:
- rounded corners used consistently,
- subtle halo, glow, texture, or panel layering,
- elegant separators,
- restrained shadows.

Avoid:
- loud glassmorphism,
- aggressive gradients everywhere,
- gimmicky blobs used without structure,
- random border radii,
- heavy drop-shadow stacks.

## CTA hierarchy rules
Every page should have one obvious primary action.
For Masoda, the global hierarchy is usually:
1. réserver un atelier,
2. collaborer,
3. découvrir les réseaux,
4. explorer le blog ou la boutique.

UI implications:
- only one dominant button style per screen section,
- secondary actions must remain visible but calmer,
- do not put five equal-weight CTAs side by side,
- primary CTA must appear early on mobile.

## Image direction
Allowed:
- real photography,
- hands, textures, materials, objects,
- workshop moments,
- art tools,
- crafted environments,
- subtle illustration if aligned.

Not allowed:
- AI-looking fake people,
- generic stock wellness women laughing at camera,
- cliché spiritual imagery,
- visuals that feel medically therapeutic or pseudo-mystical.

## Component heuristics
### Hero sections
A Masoda hero should:
- establish mood quickly,
- explain the page clearly,
- include one dominant CTA,
- avoid overlong copy.

### Cards
Cards should be:
- clean,
- curated,
- editorial rather than marketplace-like.

Do not make cards feel like discount product grids unless the page explicitly needs that.

### Navigation
Navigation should feel simple and calm.
Keep the main nav focused. Do not overload with too many top-level items.

### Footer
Footer can hold broader discovery:
- social links,
- newsletter,
- secondary pages,
- contact,
- legal links.

## Consistency checklist
Before finalising any styling or UI refactor, verify:
- colours are restrained and coherent,
- primary/secondary button styles are clear,
- spacing feels deliberate,
- headings have a clear hierarchy,
- cards and sections look like they belong to the same brand,
- mobile layout still feels elegant,
- desktop does not feel empty or stretched,
- no section looks copied from a SaaS or ecommerce template.

## Decision shortcuts
If torn between two options:
- choose the calmer one,
- choose the more editorial one,
- choose the more legible one,
- choose the one that helps booking faster,
- choose the one that looks timeless rather than trendy.

## Output expectation
When using this skill, produce changes that improve one or more of these:
- brand coherence,
- visual hierarchy,
- readability,
- conversion clarity,
- perceived quality.

Do not introduce unnecessary stylistic complexity just to make the page look “designed”.
